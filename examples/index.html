BindrJS is a lightweight data reactivity library built to handle HTML data binding, the most efficient way possible, DOM updates are as close as possible to data updates thanks to native JavaScript Proxy API

In addition to the classic HTML string interpolation, it provides the most basic data binding types out of the box and virtually any attribute to help make your HTML powerful, dynamic and with the smoothest learning curve, mirroring vanilla JavaScript and HTML as much as possible

This library is also meant to be easily adoptable by ongoing projects that are not currently using any data-binding solutions, and require the least amount of setup as possible, making it an easy pick for quick small projects or to provide a simple data binding solution to large legacy projects

In addition to the out of the box binding types, you are able to add your own binding types and take advantage of the data reactivity system however you see fit

All you need to know is how to create HTML Bind Handlers You can create them with attributes that have a colon ( : ) before them or content inside ( ${} )

For example: 

              
  new Bind({
    id: 'main',
    bind: {
      bindClass: 'bind-class',
      bindValue: 'Bind value'
    }
  });

Both the :class attribute and ${this.bindValue} will become bind handlers.

Where the :class bind handler will depend on this.bindClass and ${this.bindValue} depends on this.bindValue. And every time each handler dependency is updated, the HTML Bind Handler's will be re-computed

Install:

To install locally: 

npm i bindrjs
And add to your index.html 
 <script src="node_modules/dist/index.js"> </script>

Or add the script tag with the CDN: 

 <script src="https://unpkg.com/bindrjs@${this.latestVersion}/dist/index.js"></script>

Custom bindings:

You are able to get all the benefit from the data reactivity however you want by defining your own custom bind handlers!

CustomBindHandler is a global function added by BindrJS that you can access anywhere, it takes 2 arguments:

  • name: is the string that will decide how you bind this handler to your HTML
  • compute: is the function that will be called every time data attached to this handler changes

    • NOTES:

  • You have to make sure your custom handlers are defined before any new instance of Bind that might need them
  • You can't create custom binds which are already coded in the library

    • Event hooks

    When a new Bind instance is created there are a few life hooks you can tap on for your own needs: 

                  
  new Bind({
    /**
    * Executed when the template has been appended but binds are still not processed
    * (useful with the template property)
    */
    templateRendered() {}

    /** 
    * Executed when the template has been rendered and binds have been processed
    */
    ready() {},

    /**
    * Executed when reactive data changes
    */
    onChange(changes) {}
  });
